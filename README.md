# tickets_hunter

针对演唱会票务的高并发场景,搭的基于Redis、MySQL和Kafka的微服务架构,实现了从抢票锁座到支付出票的全链路逻辑。

- 在原始MySQL行锁的基础上,通过引入RedisSETNX解决海量MySQL行锁冲突的问题;利用RedisBitMap存储座位状态,优化前端高频拉取导致的性能开销和带宽消耗。
- 引入Kafka消息队列对下单流程进行异步削峰,提升系统的高并发抗压能力和峰值吞吐量。
- 基于RedisZSet实现秒级精度的延迟队列实现订单的超时取消;为了应对极端情况下Redis宕机导致的消息丢失问题,通过异步任务实现高效扫表补偿,保证系统的最终一致性。
- 使用DTM-Saga模式构建分布式事务,通过其子事务屏障机制解决分布式事务中的空补偿、悬挂和幂等,保证“扣款-出票”跨微服务事务的最终一致性。
